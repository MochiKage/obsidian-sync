# Obsidian Sync

PC ↔ Android 平板 Obsidian 库双向同步工具，附带 Git 版本管理和 Web 可视化面板。

> [English](README_en.md)

## 功能

| 模块 | 说明 |
|------|------|
| **双向同步** | 三向比对算法，自动检测 PC / 平板哪边有变更，决定推拉方向 |
| **冲突检测** | 两边同时修改同一文件时自动标记冲突，保留两份不丢数据 |
| **删除确认** | 检测到删除操作时在面板弹窗确认，支持确认删除 / 保留文件 / 重新推送 |
| **安全阀** | 平板剩余文件 mtime 匹配率 < 70% 时自动阻止删除，防止扫描不完整误删 |
| **Git 版本管理** | 每次同步自动 commit（含前置快照），完整记录每条笔记的修改历史 |
| **Web 面板** | 本地浏览器操作面板，查看文件状态、变更历史、diff 对比 |
| **暂存区** | 选择性同步——勾选文件加入暂存区，只同步选中的文件 |
| **冲突合并** | 面板中可视化查看冲突，一键选择保留 PC 版或平板版 |
| **变更搜索** | 按关键词搜索所有历史 diff，追溯每次修改 |
| **健康检查** | 检测孤立附件、断链 wikilink、意外入库的大文件 |
| **文件夹切换** | 面板内自由浏览和切换 PC 端库文件夹 |
| **错误日志** | 所有异常写入 `.sync_errors.log`，带时间戳和完整堆栈 |
| **异常处理** | USB 断开快速失败（~5s），执行中连续失败自动跳过，syncstate 原子写入防崩坏 |

## 环境要求

- **Python** 3.8+
- **Git**
- **ADB**（Android Debug Bridge）—— 平板需开启 USB 调试
- **Flask**（`pip install flask`）

## 快速开始

### 1. 项目结构

```
D:\Sekai\obsidian-sync\
├── sync.py            # 同步引擎（Config / ADB / SyncState / GitManager / ObsidianSync）
├── dashboard.py       # Flask Web 面板（API 路由 + 后台线程 + 错误处理）
├── config.json        # 用户配置
├── start.bat          # 一键启动脚本
├── .sync_errors.log   # 错误日志（自动生成）
├── templates/
│   └── index.html     # 单页三栏布局 + 弹窗
└── static/
    ├── style.css      # 暗色主题样式
    └── app.js         # 前端交互逻辑
```

### 2. 配置

编辑 `config.json`：

```json
{
  "pc_vault_path": "D:/Documents/Obsidian/MyVault",
  "tablet_vault_path": "/sdcard/Documents/Note",
  "adb_path": "auto",
  "conflict_strategy": "keep_both",
  "delete_strategy": "propagate",
  "ignore_patterns": [
    ".git/",
    ".syncstate.json",
    ".obsidian/workspace*.json",
    ".obsidian/app.json",
    ".obsidian/appearance.json",
    ".trash/",
    ".DS_Store"
  ]
}
```

| 字段 | 说明 |
|------|------|
| `pc_vault_path` | PC 端 Obsidian 库路径，支持在面板中切换 |
| `tablet_vault_path` | 平板端库路径，通常是 `/sdcard/Documents/Note` |
| `adb_path` | ADB 路径，`"auto"` 自动检测（winget / C:\platform-tools / SDK / PATH） |
| `conflict_strategy` | 冲突策略：`"keep_both"` 保留两份 / `"skip"` 跳过 |
| `delete_strategy` | 删除策略：`"propagate"` 同步删除 / `"ignore"` 忽略 |
| `ignore_patterns` | 不同步的文件模式（glob 格式，`/` 结尾 = 整个目录） |

> ⚠️ 建议 `delete_strategy` 设为 `"ignore"`，避免平板端文件意外丢失时连带删除 PC 端。删除操作在面板中有确认弹窗。

### 3. 连接平板

1. 平板开启 **USB 调试**（设置 → 开发者选项）
2. 用 USB 线连接电脑
3. 平板上弹出的 RSA 指纹对话框点「允许」
4. 终端验证：`adb devices` 应显示设备

### 4. 初始化

```bash
python sync.py --init
```

这会：
- 创建 PC 端库文件夹
- 初始化 Git 仓库
- 扫描平板端现有文件，建立同步状态基线

### 5. 日常使用

```bash
# Web 面板模式（推荐）
python dashboard.py         # 或双击 start.bat
# 浏览器打开 http://localhost:8820

# 命令行模式
python sync.py              # 执行同步
python sync.py --dry-run    # 预览变更（不执行）
```

## 同步算法

三向比对（three-way comparison）：

```
输入: PC 快照 + 平板快照 + 上次同步状态 (.syncstate.json)

生命周期: 前置快照 → 扫描 → 比对 → [确认删除] → 执行 → 重新扫描 → 后置提交

对每个文件：
  ├─ 两边都有
  │   ├─ 都不变      → 跳过
  │   ├─ 只有 PC 变  → PUSH（推送到平板）
  │   ├─ 只有平板变  → PULL（拉取到 PC）
  │   └─ 两边都变    → CONFLICT（保留两份）
  │
  ├─ 只有 PC 有
  │   ├─ 平板曾有此文件 → 平板端被删 → 按删除策略处理
  │   └─ 纯新增        → PUSH
  │
  ├─ 只有平板有
  │   ├─ PC 曾有此文件  → PC 端被删  → 按删除策略处理
  │   └─ 纯新增        → PULL
  │
  └─ 都没有但有记录 → 两边都删了 → 静默清除
```

### 安全阀

删除操作 ≥ 5 个时，检查平板剩余文件的 mtime 匹配率：
- 匹配率 ≥ 70% → 扫描准确，删除是真操作 → **放行**
- 匹配率 < 70% → 扫描可疑 → **阻止删除并警告**

## Git 版本管理

PC 端库自动纳入 Git 管理，每次同步产生 1~2 个 commit：

```
523635c init: initial vault snapshot
ebc11b4 pre-sync snapshot: 2026-06-25 17:57:58   ← 同步前快照
5eb72ba sync: pushed 1                            ← 同步结果（含 del-pc/del-tab）
```

- **pre-sync snapshot**：同步前自动保存当前 PC 状态，就算同步出错也能回滚
- **sync**：记录本次同步的变更概要（推拉数、冲突数、删除数）

查看历史：

```bash
cd <你的库目录>
git log --oneline          # 查看历史
git diff HEAD~1            # 查看最近一次同步的变更
git show <commit>          # 查看某次提交的完整 diff
```

## 异常处理

| 场景 | 行为 |
|------|------|
| **USB 断开（扫描前）** | 5s 快速 ADB 复检，断开立即报错 |
| **USB 断开（执行中）** | 连续 2 次 push/pull 失败后跳过剩余 ADB 操作 |
| **程序被杀** | `.syncstate.json` 先写 `.tmp` 再 rename，不会半写崩坏；`.tmp` 存在则自动恢复 |
| **平板扫描不完整** | 安全阀阻止批量删除 |
| **所有异常** | 写入 `.sync_errors.log`，含时间戳和完整堆栈 |

## 性能

| 优化 | 说明 |
|------|------|
| 平板扫描合并 | `find` + `stat` + `md5sum` 合并为 1 次 adb shell 调用 |
| 扫描缓存 | 30s TTL，页面内多次请求复用结果 |
| 暂存操作 | 不触发文件重扫描，瞬间完成 |
| 健康检查缓存 | 60s 缓存，避免反复读取全部 .md 文件 |

## 冲突处理

当 PC 和平板同时修改了同一文件：

1. **命令行模式**：PC 版本保留原名，平板版本保存为 `文件名.conflict.md`
2. **Web 面板**：冲突文件显示「保留 PC 版本」「保留平板版本」按钮，一键解决
